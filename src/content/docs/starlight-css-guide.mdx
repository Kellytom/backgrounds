# Starlight CSS Customization Guide

This comprehensive guide covers how to customize **Starlight** with CSS, including theme variables, component overrides, and advanced styling techniques used in the **Hacker1** project.

## CSS Architecture

Starlight uses a layered CSS architecture that allows for flexible customization:

```
1. Base Starlight Styles (Framework)
2. Theme Variables (Colors, fonts, spacing)
3. Component Overrides (Custom styling)
4. Global Customizations (Site-wide changes)
```

## Theme Variables

Starlight uses CSS custom properties for theming. Here are the key variables you can customize:

### Color Variables

```css
:root {
  /* Primary brand colors */
  --sl-color-accent: hsl(136, 58%, 40%);
  --sl-color-accent-low: hsl(136, 54%, 15%);
  --sl-color-accent-high: hsl(136, 58%, 90%);
  
  /* Text colors */
  --sl-color-text: hsl(224, 9%, 86%);
  --sl-color-text-invert: hsl(224, 13%, 12%);
  --sl-color-text-accent: hsl(136, 58%, 55%);
  
  /* Background colors */
  --sl-color-bg: hsl(224, 13%, 12%);
  --sl-color-bg-nav: hsl(224, 13%, 10%);
  --sl-color-bg-sidebar: hsl(224, 13%, 10%);
  
  /* Border colors */
  --sl-color-gray-1: hsl(224, 7%, 25%);
  --sl-color-gray-2: hsl(224, 9%, 31%);
  --sl-color-gray-3: hsl(224, 9%, 42%);
  --sl-color-gray-4: hsl(224, 9%, 56%);
  --sl-color-gray-5: hsl(224, 9%, 73%);
  --sl-color-gray-6: hsl(224, 10%, 88%);
}

/* Light theme overrides */
[data-theme='light'] {
  --sl-color-bg: hsl(0, 0%, 100%);
  --sl-color-bg-nav: hsl(224, 13%, 97%);
  --sl-color-text: hsl(224, 13%, 12%);
  --sl-color-text-invert: hsl(224, 9%, 86%);
}
```

## Component Styling

### Custom Navbar (Hacker1 Example)

Our custom navbar overrides the default Starlight header:

```css
/* Custom navbar styling */
.custom-navbar {
  background: #1a1a1a;
  padding: 1rem 0;
  border-bottom: 1px solid #333;
  position: relative;
  z-index: 1000;
}

.navbar-container {
  max-width: 1200px;
  margin: 0 auto;
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 0 1rem;
  gap: 2rem;
}

/* Hide default Starlight header */
:global(.header) {
  display: none;
}
```

### Search Component Customization

Override Starlight's search styling:

```css
/* Search component overrides */
.search-container :global(.search) {
  background: rgba(255, 255, 255, 0.2);
  border: 1px solid #666;
  border-radius: 6px;
  position: relative;
  z-index: 1001;
}

.search-container :global(.search input) {
  background: transparent;
  color: #fff;
  border: none;
}

.search-container :global(.search input::placeholder) {
  color: #bbb;
}

/* Search results dropdown */
.search-container :global(.search-results) {
  background: var(--sl-color-bg-nav);
  border: 1px solid var(--sl-color-gray-2);
  border-radius: var(--sl-border-radius);
  box-shadow: var(--sl-shadow-lg);
}
```

## Advanced Customization

### Custom Starlight Components

Override specific Starlight components by creating files in `src/components/`:

```astro
---
// src/components/Header.astro
import Navbar from './Navbar.astro';
---

<!-- Replace default header with custom navbar -->
<Navbar />

<style>
/* Additional header styling */
</style>
```

### Theme-Specific Styling

Create styles that respond to theme changes:

```css
/* Base styles */
.my-component {
  color: var(--sl-color-text);
  background: var(--sl-color-bg);
}

/* Dark theme specific */
[data-theme='dark'] .my-component {
  border-color: var(--sl-color-gray-2);
}

/* Light theme specific */
[data-theme='light'] .my-component {
  border-color: var(--sl-color-gray-4);
  box-shadow: var(--sl-shadow-sm);
}
```

### Responsive Design

Starlight includes responsive breakpoints:

```css
/* Starlight breakpoints */
@media (min-width: 50rem) {
  /* Tablet and up */
  .responsive-component {
    display: grid;
    grid-template-columns: 1fr 3fr;
  }
}

@media (min-width: 72rem) {
  /* Desktop and up */
  .responsive-component {
    grid-template-columns: 1fr 2fr 1fr;
  }
}
```

## Global Styling

### Site-wide CSS

Add global styles to your Astro layout or component:

```css

/* Global resets and improvements */
*,
*::before,
*::after {
  box-sizing: border-box;
}

/* Smooth scrolling */

  scroll-behavior: smooth;
}

/* Focus styles */
:focus-visible {
  outline: 2px solid var(--sl-color-accent);
  outline-offset: 2px;
}

/* Selection styles */
::selection {
  background: var(--sl-color-accent-low);
  color: var(--sl-color-accent-high);
}
```

## Performance Optimization

### CSS Organization

Organize your CSS for better maintainability:

```css
/* 1. Variables and tokens */
:root { /* custom properties */ }

/* 2. Base and reset styles */
*, *::before, *::after { /* reset */ }

/* 3. Component styles */
.component { /* component-specific */ }

/* 4. Utility classes */
.sr-only { /* utilities */ }

/* 5. Theme overrides */
[data-theme="dark"] { /* theme-specific */ }

/* 6. Media queries */
@media (min-width: 50rem) { /* responsive */ }
```

### Critical CSS

Inline critical CSS for above-the-fold content:

```astro
---
// In your layout file
---

<style is:inline>
/* Critical styles for immediate rendering */
.custom-navbar {
  background: #1a1a1a;
  /* ... other critical navbar styles */
}
</style>

<style>
/* Non-critical styles */
.search-container :global(.search-results) {
  /* ... dropdown styles that aren't immediately visible */
}
</style>
```

## Best Practices

1. **Use CSS Custom Properties**: Always prefer custom properties over hardcoded values
2. **Scope Styles**: Use component-scoped styles to avoid conflicts
3. **Follow Starlight Patterns**: Maintain consistency with Starlight's design system
4. **Test Themes**: Always test custom styles in both light and dark themes
5. **Performance**: Use efficient selectors and minimize CSS bundle size
6. **Accessibility**: Ensure custom styles don't break accessibility features
